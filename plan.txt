Дополнительно:
- Перевод должен происходить в том же файле, в котором находится оригинал
- Нельзя менять уже переведенные части
- Нельзя менять переводы в глоссарии если они уже были переведены
- Задача может быть выполнена в несколько итераций. При необходимости указать в файле задачи номер итерации.
- Использовать файлы задач для помощи в переводе и контроля выполнения задач
- Регулярно обновлять глоссарий новыми терминами
- Если в процессе исполнения задачи возникают вопросы или сложности, то указать в файле задачи
- Если не хватает информации для выполнения задачи, то указать в файле задачи

Правила работы с задачами:

1. Создание задачи
   - Сформулировать четкое описание задачи
   - Создать файл задачи с именем task_[ID].md
   - Сформулировать четкое описание задачи в файле

2. Планирование
   - Составить план исполнения задачи
   - Добавить чекбоксы для каждого шага плана

3. Исполнение
   - Выполнить все шаги плана
   - Обновить файл задачи с результатами выполнения
   - Обновить все затронутые файлы


4. Проверка
   - Заполнить чекбоксы завершенных шагов
   - Убедиться, что все чекбоксы отмечены
   - Проверить полноту выполнения задачи
   - Сделать коммит

5. Коммит
   - Подготовить описание выполненной работы
   - В описании коммита должно быть указано ID задачи и название задачи
   - Добавить в stage файлы, которые были изменены в результате выполнения задачи
   - Использовать semantic commit для сообщения коммита:
       - feat: для новых функций
       - fix: для исправления ошибок
       - docs: для обновления документации
       - style: для изменений форматирования
       - refactor: для рефакторинга кода
       - test: для добавления тестов
       - chore: для обновления зависимостей и т.п.

6. Закрытие задачи
   - Убедиться, что все предыдущие шаги выполнены
   - Закрыть задачу в системе управления задачами
